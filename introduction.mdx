Dev0 is an agentic engineering platform inspired by the Ralph loop: let a code agent run until the task is actually done, but keep it contained, auditable, and reviewable. You describe a product idea, Dev0 plans the work, executes tasks inside secure E2B sandbox environments, and opens a GitHub pull request for each one. You stay in control as the technical lead while the platform handles execution.

## Who it's for

- **Solo engineers** — ship features faster by delegating implementation to agents while you stay in the review seat.
- **Startups** — reduce build time with a PR-first automation pipeline that keeps your codebase clean.
- **Teams** — use review as the control plane for all AI execution, so every change is traceable.
- **Hackathon teams** — turn rough ideas into working, committed code with live progress you can watch.

## Prerequisites

Before you run Dev0, make sure you have the following:

| Requirement | Purpose |
|---|---|
| [Bun](https://bun.sh) | Runtime and package manager |
| PostgreSQL | Application database |
| [E2B API key](https://e2b.dev) | Sandbox execution environments |
| Google Gemini API key | Planning and preview generation (Vercel AI SDK) |
| Agent Gemini API key | Autonomous coding inside sandboxes (YOLO mode) |
| Upstash Redis credentials | Realtime pub/sub for log streaming |
| GitHub token | Repository creation and PR operations |

<Note>
  You need two separate Gemini API keys: one for the server-side Vercel AI SDK calls (`GOOGLE_GENERATIVE_AI_API_KEY`) and one injected into sandboxes for the Gemini CLI agent (`AGENT_GEMINI_API_KEY`). They can point to the same Google Cloud project but must be configured separately.
</Note>
