Sandboxes are isolated E2B execution environments provisioned per project. Each task execution runs inside a sandbox that has the project's GitHub repository cloned and the appropriate tech stack installed.

## Sandbox schema

The `sandboxes` table is defined in `src/lib/db/schema.ts`.

```typescript
type Sandbox = InferSelectModel<typeof sandboxes>
```

### Fields

<ResponseField name="id" type="string" required>
  UUID primary key. Auto-generated by the database.
</ResponseField>

<ResponseField name="projectId" type="string" required>
  UUID of the parent project. Foreign key to `projects.id`. Deleting the project cascades to delete all sandbox records (`CASCADE`).
</ResponseField>

<ResponseField name="sandboxId" type="string" required>
  The E2B sandbox identifier returned by the E2B API. Used in all subsequent E2B operations (execute command, read file, etc.). Unique across all rows.
</ResponseField>

<ResponseField name="status" type="SandboxStatus" required>
  Current state of the sandbox. Default: `'READY'`.

  <Expandable title="status values">
    <ResponseField name="READY" type="string">
      The sandbox is provisioned and idle. Ready to accept the next task.
    </ResponseField>
    <ResponseField name="RUNNING" type="string">
      A task is currently executing inside this sandbox.
    </ResponseField>
    <ResponseField name="STOPPED" type="string">
      The sandbox has been stopped. It may be resumed or a new one provisioned.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="taskId" type="string | null">
  UUID of the task currently (or most recently) running in this sandbox. Foreign key to `tasks.id`. Unique — only one task per sandbox at a time.
</ResponseField>

<ResponseField name="snapshotId" type="string | null">
  E2B snapshot ID used to resume the sandbox from a known state, avoiding a full cold-start.
</ResponseField>

<ResponseField name="publicUrl" type="string | null">
  Public HTTPS URL exposed by E2B for the dev server running inside the sandbox. Set by `startProjectPreview` and used by the preview pane.
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp set at insert time.
</ResponseField>

<ResponseField name="updatedAt" type="string" required>
  ISO 8601 timestamp updated automatically on every row change.
</ResponseField>

## Status enum

```typescript
type SandboxStatus = 'READY' | 'RUNNING' | 'STOPPED'
```

## Relations

```typescript
export const sandboxRelations = relations(sandboxes, ({ one }) => ({
  project: one(projects, {
    fields: [sandboxes.projectId],
    references: [projects.id],
  }),
  task: one(tasks, {
    fields: [sandboxes.taskId],
    references: [tasks.id],
  }),
}))
```

| Relation | Type | Description |
|---|---|---|
| `project` | `Project` | The project this sandbox belongs to. |
| `task` | `Task \| null` | The task currently assigned to this sandbox. |

## TaskExecutionLog model

Execution logs are stored in a separate `task_logs` table and loaded lazily to avoid inflating task list payloads.

```typescript
type TaskExecutionLog = InferSelectModel<typeof taskLogs>
```

### Fields

<ResponseField name="id" type="string" required>
  UUID primary key.
</ResponseField>

<ResponseField name="taskId" type="string" required>
  UUID of the parent task. Foreign key to `tasks.id`. One-to-one — each task has at most one log record. Deleting the task cascades to delete the log.
</ResponseField>

<ResponseField name="events" type="GeminiStreamEvent[]" required>
  Array of structured events emitted by `gemini --output-format stream-json`. Each element is a `GeminiStreamEvent`. Default: `[]`.

  See [GeminiStreamEvent types](#geministreamevents) below.
</ResponseField>

<ResponseField name="summary" type="string | null">
  The last assistant message from the session, truncated to 2000 characters. Shown in the task list as a quick summary of what the agent did.
</ResponseField>

<ResponseField name="totalTokens" type="number | null">
  Total token count from the `result` event stats (`input_tokens + output_tokens`).
</ResponseField>

<ResponseField name="durationMs" type="number | null">
  Wall-clock duration of the agent execution in milliseconds.
</ResponseField>

<ResponseField name="toolCallsCount" type="number" required>
  Number of `tool_use` events recorded during the session. Default: `0`.
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp set at insert time.
</ResponseField>

<ResponseField name="updatedAt" type="string" required>
  ISO 8601 timestamp updated automatically on every row change.
</ResponseField>

## GeminiStreamEvent types

The `events` column in `task_logs` stores an array of `GeminiStreamEvent` objects. Each event is emitted by the Gemini CLI in JSONL format (`--output-format stream-json`).

```typescript
type GeminiStreamEvent =
  | GeminiInitEvent
  | GeminiMessageEvent
  | GeminiToolUseEvent
  | GeminiToolResultEvent
  | GeminiErrorEvent
  | GeminiResultEvent
```

All events share a `timestamp` field:

```typescript
interface GeminiBaseEvent {
  timestamp: string // ISO 8601
}
```

### GeminiInitEvent

Emitted once at the start of the session.

```typescript
interface GeminiInitEvent extends GeminiBaseEvent {
  type: 'init'
  session_id: string
  model: string
}
```

### GeminiMessageEvent

Emitted for user prompts and assistant responses. Incremental streaming updates have `delta: true`.

```typescript
interface GeminiMessageEvent extends GeminiBaseEvent {
  type: 'message'
  role: 'user' | 'assistant'
  content: string
  delta?: boolean
}
```

<Note>
  User message events (`role: 'user'`) are filtered out before the logs are returned to the client by `getTaskWithLogs`. They are stored in the database but not exposed via the API.
</Note>

### GeminiToolUseEvent

Emitted when the agent invokes a tool (e.g. read a file, run a command).

```typescript
interface GeminiToolUseEvent extends GeminiBaseEvent {
  type: 'tool_use'
  tool_name: string
  tool_id: string
  parameters: Record<string, JsonValue>
}
```

### GeminiToolResultEvent

Emitted with the result of a tool call.

```typescript
interface GeminiToolResultEvent extends GeminiBaseEvent {
  type: 'tool_result'
  tool_id: string
  status: 'success' | 'error'
  output?: string
  error?: string
}
```

### GeminiErrorEvent

Emitted for non-fatal errors during the session.

```typescript
interface GeminiErrorEvent extends GeminiBaseEvent {
  type: 'error'
  message: string
  code?: string
}
```

### GeminiResultEvent

Final event emitted after the CLI exits. Contains aggregated usage statistics.

```typescript
interface GeminiResultEvent extends GeminiBaseEvent {
  type: 'result'
  status: 'success' | 'error'
  stats: GeminiResultStats
}

interface GeminiResultStats {
  total_tokens?: number
  input_tokens?: number
  output_tokens?: number
  duration_ms?: number
  tool_calls?: number
}
```
