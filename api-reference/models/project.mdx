title: "Project model"

Projects are the top-level entity in Dev0. A project contains a GitHub repository, a set of tasks, and a specification generated by the AI planner.

## Schema

The `projects` table is defined using Drizzle ORM in `src/lib/db/schema.ts`.

```typescript
type Project = InferSelectModel<typeof projects>
```

## Fields

<ResponseField name="id" type="string" required>
  UUID primary key. Auto-generated by the database (`uuid_generate_v4()`).
</ResponseField>

<ResponseField name="name" type="string" required>
  Human-readable project name. Set at creation time and not updated automatically.
</ResponseField>

<ResponseField name="description" type="string | null">
  Short description of the project. May be `null` if not provided at creation.
</ResponseField>

<ResponseField name="status" type="ProjectStatus" required>
  Current lifecycle status of the project. Default: `'PLANNING'`.

  <Expandable title="status values">
    <ResponseField name="PLANNING" type="string">
      Initial state. The AI planner is generating the spec and task list. The GitHub repository does not yet exist.
    </ResponseField>
    <ResponseField name="READY" type="string">
      Planning complete. The repository is initialised and tasks are ready to execute.
    </ResponseField>
    <ResponseField name="ACTIVE" type="string">
      One or more tasks are currently running or recently ran.
    </ResponseField>
    <ResponseField name="PAUSED" type="string">
      Execution is paused. No tasks are running.
    </ResponseField>
    <ResponseField name="COMPLETED" type="string">
      All tasks are done. The project is considered finished.
    </ResponseField>
    <ResponseField name="ARCHIVED" type="string">
      The project has been archived. It is read-only.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="repoUrl" type="string | null">
  Full HTTPS URL of the GitHub repository (e.g. `https://github.com/org/repo`). Set after the repository is created. `null` while the project is in `PLANNING` status.
</ResponseField>

<ResponseField name="repoName" type="string | null">
  Short repository name without the owner prefix (e.g. `my-app`). Used to construct API calls to GitHub. `null` while the project is in `PLANNING` status.
</ResponseField>

<ResponseField name="techStack" type="string" required>
  Identifier for the frontend framework template. Currently always set to `'react-vite'`. Default: `'react-vite'`.
</ResponseField>

<ResponseField name="theme" type="string" required>
  UI color theme key used when scaffolding the project. Default: `'slate'`.
</ResponseField>

<ResponseField name="vibeInput" type="string | null">
  The original free-form natural language input the user provided when creating the project. Stored for reference and replay.
</ResponseField>

<ResponseField name="specContent" type="string | null">
  AI-generated Markdown project specification. Includes name, tagline, overview, feature list, and optional technical notes. Injected into each agent task prompt. `null` until planning completes.
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp. Set by the database at insert time.
</ResponseField>

<ResponseField name="updatedAt" type="string" required>
  ISO 8601 timestamp. Automatically updated by the database on every row change.
</ResponseField>

## Status enum

```typescript
type ProjectStatus =
  | 'PLANNING'
  | 'READY'
  | 'ACTIVE'
  | 'PAUSED'
  | 'COMPLETED'
  | 'ARCHIVED'
```

The database enforces this as a PostgreSQL `pgEnum`. Use `updateProjectStatus` to transition between states.

---

## Relations

A project has the following Drizzle relations:

```typescript
export const projectRelations = relations(projects, ({ many }) => ({
  tasks: many(tasks),
  sandboxes: many(sandboxes),
}))
```

| Relation | Type | Description |
|---|---|---|
| `tasks` | `Task[]` | All tasks belonging to this project. Deleted when the project is deleted (`CASCADE`). |
| `sandboxes` | `Sandbox[]` | All E2B sandbox records provisioned for this project. Deleted when the project is deleted (`CASCADE`). |
